Admin training guides

Administer Dove365 CRM

Use this guide to set up and manage the Dove365 environment. It covers user and role management, mailbox configuration, SharePoint document integration, service offerings, and Dynamics 365 App for Outlook deployment and usage.

Admin overview

What the Dove365 Admin app does

The Dove365 Admin app is part of Dove365 Common and provides the administration layer for the Power Platform environment. It is separate from the CRM Starter app and requires the Dove365 Global Admin security role to access.

Overview

Start here if you are setting up the environment for the first time or taking over administration.

Admin app areas

  • User Management
  • SharePoint Settings
  • Data Configuration
  • Support / Training

Required role

  • Dove365 Global Admin

1Assign Dove365 Global Admin

Before accessing the Admin app, ensure your account has the Dove365 Global Admin security role assigned in the Dataverse environment. This is done from Settings > Security > Users in the Power Platform admin centre or classic advanced settings.

2Open the Dove365 Admin app

Navigate to the Dove365 Admin model-driven app from the app picker. The navigation shows User Management, SharePoint Settings, Data Configuration, and Support.

3Follow the setup sequence

Work through Users, Teams, Positions, Mailboxes, SharePoint Settings, and Service Offerings in that order for a clean first-time setup.

Dove365 Common must be installed before the Admin app is available. CRM Starter is a separate solution installed after Common.

Users

Manage user records, assign security roles, and configure organisation details. Users must already exist in the Dataverse environment before appearing here.

Key fields

  • Full Name and User Name
  • Job Title and Primary Email
  • Mobile and Main Phone
  • Business Unit and Manager
  • Position
  • Security roles (via PCF control)

1Locate the user

Open User Management > Users. Users are provisioned through the Microsoft 365 admin centre and appear in Dataverse after licensing and environment access is granted. Use search or the active users view to find the user.

2Update profile details

Open the user record and complete Job Title, Primary Email, Mobile Phone, and Main Phone. These fields populate CRM views and activity tracking.

3Set Business Unit, Manager, and Position

Assign the correct Business Unit, Manager lookup, and Position to place the user in the right part of the reporting hierarchy.

4Assign Dove365 security roles using the Security Role Manager

Scroll to the Security section on the user form. The Security Role Manager PCF control shows all available Dove365 security roles as checkboxes. Check the roles you want to assign. Changes apply immediately — no form save is needed for role changes.

If the environment has Dynamics 365 enabled, the Dynamics 365 for Outlook User role will also appear in the Security Role Manager. Check it to enable App for Outlook access for that user.

5Check the Mailbox record

After setting up the user, go to User Management > Mailboxes and confirm the user's mailbox is listed. Follow the Mailboxes guide to approve and enable it.

Standard roles to know

  • Dove365 CRM User — standard CRM access for day-to-day users
  • Dove365 CRM Admin — administrative CRM access including service offering maintenance
  • Dove365 Global Admin — full admin app access
  • Dynamics 365 for Outlook User — enables the Outlook add-in where applicable

Teams

Teams group users for role assignment, record ownership, and hierarchy. Security roles assigned to a team apply to all members of that team.

Key fields

  • Team Name
  • Business Unit
  • Administrator
  • Team Type
  • Membership Type
  • Members (subgrid)
  • Security roles (via PCF control)

1Create a team

Open User Management > Teams and select New. Enter the Team Name, assign a Business Unit, and choose an Administrator. Set Team Type to Owner for standard teams.

2Add team members

Use the Team Members subgrid on the team form to add users. Members inherit the roles assigned to the team.

3Assign security roles to the team

Scroll to the Security section on the team form. The Security Role Manager PCF control lists all available Dove365 roles. Check the roles you want all team members to have.

Assigning a role to a team does not automatically propagate it to individual user records. Users will have the role through team membership, but it will not appear checked on their individual user Security Role Manager unless also directly assigned.

4Use Microsoft Entra ID (AAD) groups if applicable

If your organisation uses Azure Active Directory groups, you can create AAD group teams in Dataverse. Set the Team Type to AAD Security Group and provide the Object ID from Entra ID. Membership is managed through the group rather than manually.

Positions

Positions define the reporting hierarchy for your organisation. They are assigned to users to reflect their place in the structure.

Use positions for

  • Defining seniority levels
  • Structuring reporting lines
  • Supporting hierarchy-based record access

1Create positions

Open User Management > Positions and select New. Enter a descriptive name for each position in your hierarchy, such as Account Manager, Senior Consultant, or Operations Director.

2Set parent positions

Where positions have a reporting line, set the Parent Position lookup to reflect the hierarchy. This allows Dataverse to apply position-based record access rules if configured in security role definitions.

3Assign positions to users

Return to the User form and set the Position field for each user to the appropriate position in the hierarchy.

Positions are used by Dataverse hierarchy-based access rules. If your security roles do not use hierarchy-based access, positions are still useful for organisation reporting and directory purposes.

Mailboxes

Mailboxes must be approved and tested before email tracking, server-side synchronisation, and Dynamics 365 App for Outlook will work for a user.

Mailbox status fields

  • Incoming Email Status
  • Outgoing Email Status
  • Appointments, Contacts, and Tasks Status
  • Email Address Approved By
  • Email Address Approved On
  • Test Completed On

1Open User Management > Mailboxes

Use the Active Mailboxes view to see all mailboxes in the environment. Each user and the service account should have a mailbox record.

2Check the server profile

Open a mailbox record and confirm the Server Profile is set to the correct Exchange/Microsoft 365 email server profile. Incoming Email, Outgoing Email, and Appointments, Contacts, and Tasks synchronisation methods should all be set to Server-Side Synchronization.

3Approve the email address

Select the mailbox record, then from the command bar select Approve Email and confirm. This approves the email address for use with server-side sync. An O365 admin approval may also be required depending on your Exchange configuration.

4Test and enable the mailbox

After approval, select Test & Enable Mailbox from the command bar. The system will run a configuration test. Wait for the test to complete and check the Configuration Status section — Incoming Email Status, Outgoing Email Status, and ACT Status should all show Success.

5Resolve errors

If any status shows Failure, check the error message on the mailbox record. Common causes include the email not being approved, the server profile missing, or OAuth not being enabled on Exchange. Correct the issue and re-run Test & Enable.

6Enable the service account mailbox

The Dove365 service account used by Power Automate connection references must also have its mailbox approved and tested. Ensure the Outlook Connect Service Account connection reference uses this account and is connected.

A mailbox can only be synchronised with one Dataverse environment at a time. If a user's mailbox was previously connected to another environment, the setting in Exchange must be overwritten before it will work in this one.

SharePoint Settings

SharePoint Settings lets admins configure and review document locations linked to Dataverse records, and browse documents stored in those locations.

SharePoint Settings includes

  • Document Locations
  • Documents (browse view)

Prerequisites

  • Server-based SharePoint integration enabled
  • SharePoint site URL configured
  • Document Management Settings set for required tables

1Enable server-based SharePoint integration

Before document locations will work, SharePoint integration must be enabled from the Power Platform admin centre or from Settings > Document Management in Advanced Settings. Provide the SharePoint site URL and complete the wizard.

2Configure Document Management Settings

In Advanced Settings > Document Management > Document Management Settings, select the tables (such as Account, Contact, Opportunity, Case) for which SharePoint document folders should be created. Save the settings.

3Review Document Locations

Open SharePoint Settings > Document Locations in the Admin app. This view shows all configured document locations linked to Dataverse records. Each record with document management enabled gets a document location pointing to its SharePoint folder.

4Browse Documents

Open SharePoint Settings > Documents to browse files stored in managed document locations. This is a read-only review view for admins to confirm integration is working and files are being stored correctly.

5Troubleshoot document issues

If document tabs are not showing on records, confirm: server-based SharePoint integration is enabled; Document Management Settings include that table; the SharePoint site is reachable; and the user has sufficient SharePoint permissions to the document library.

Contact records require SharePoint document management specifically enabled for the Contact table in Document Management Settings. Check this first if contact documents are not appearing.

Service Offerings

Service Offerings are the central service catalogue used by CRM users when attaching services to opportunities and cases. Admins own and maintain this catalogue.

Key fields

  • Service Name and Code
  • Service Category
  • Service Type
  • Billing Type
  • Default Price, Cost, Duration
  • Currency
  • Description

1Open Data Configuration > Service Offerings

Navigate to this area in the Admin app. This is the same Service Offering table used across the CRM, but managed here by admins rather than users.

2Create a service offering

Select New and complete the required fields. At minimum, provide a Service Name, Service Category, and Service Type. A meaningful Service Code helps with reporting and filtering.

  • Service NameClear, consistent name visible to all CRM users when selecting services.
  • Service CodeShort reference code for internal reporting and categorisation.
  • Service CategoryCategory from the global choice — e.g. Consulting, IT & Technology, Sales, Support.
  • Service TypeType of service — One-Time, Recurring, Project, or Subscription.
  • Billing TypeHow the service is billed — Fixed, Time & Materials, Retainer, etc.
  • Default Price / CostReference values for pipeline context and reporting. Not a full invoicing module.
  • DurationExpected service duration and unit (hours, days, weeks, months).

3Keep the catalogue consistent

Agree on naming conventions before creating records. Inconsistent names lead to fragmented reporting. Avoid duplicates by searching before creating new offerings.

4Deactivate rather than delete

If a service offering is no longer used, deactivate it rather than deleting it. Deleting will break existing opportunity and case records that reference the offering.

Service Offering pricing fields are for CRM reference and pipeline context only. They are not part of a quoting, ordering, or invoicing module.

Dynamics 365 App for Outlook — Admin Setup

Deploy Dynamics 365 App for Outlook so users can track emails, view CRM data, and create records directly from Outlook on desktop and web.

Prerequisites

  • Server-side synchronisation configured
  • Mailboxes approved and tested
  • OAuth enabled on Exchange
  • Dynamics 365 for Outlook User role assigned

Supported clients

  • Outlook desktop (Windows)
  • Outlook Web Access
  • Outlook mobile (iOS / Android)

1Confirm prerequisites

Before deploying, ensure server-side synchronisation is enabled and mailboxes are approved and tested for the target users. The add-in requires Exchange Web Services (EWS) and OAuth to function. Refer to the Mailboxes section of this guide for setup steps.

2Assign the Dynamics 365 for Outlook User role

In the Dove365 Admin app, open User Management > Users and locate the target user. In the Security section, use the Security Role Manager control to check Dynamics 365 for Outlook User. This grants the privileges needed to access the add-in.

Assigning the role to a team does not automatically grant App for Outlook privileges to team members. Assign the role directly to individual user records for App for Outlook deployment.

3Deploy the app to users

From Advanced Settings in the CRM or Power Platform, go to Settings > Dynamics 365 App for Outlook. The Getting Started page lists all eligible users. You have three deployment options:

  • Auto deploy to all eligible users — enable the checkbox at the top. Users with server-side sync configured are added automatically.
  • Add for all eligible users now — select Add App for Eligible Users to push immediately.
  • Add for selected users — select specific users in the list and choose Add App to Outlook.

Status changes from Pending to Added to Outlook when successful. The add-in then appears in the Outlook ribbon for those users.

4Allow self-install (optional)

Users can also install the add-in themselves. Direct them to Settings > Apps for Dynamics 365 apps in their CRM environment, where they can select Add app to Outlook without admin intervention — provided they have the security role and a working mailbox.

5Troubleshoot installation issues

If a user cannot install the add-in, check: the Dynamics 365 for Outlook User role is assigned directly to the user; their mailbox is approved and tested; their Exchange mailbox is not already linked to a different Dataverse environment. An Exchange mailbox can only synchronise with one Dataverse environment at a time.

Both steps are required — assigning the security role alone does not make the add-in available. The role plus a working mailbox and either admin deployment or self-install are all needed.

Dynamics 365 App for Outlook — User Guide

Once deployed, users can open the Dynamics 365 pane in Outlook to track emails, link records, view CRM data, and create records without leaving Outlook.

Key actions

  • Open the Dynamics 365 pane
  • Sign in to your environment
  • Track an email
  • Set Regarding to link a record
  • View contact and account details
  • Create a new record
  • Pin the pane open

Terminology

  • Track — copy the email into Dataverse
  • Set Regarding — link the email to a specific record
  • Untrack — remove the copy from Dataverse

1Open the Dynamics 365 pane

In Outlook desktop, select an email and choose Dynamics 365 from the ribbon. In Outlook on the web, open an email and select More (...) > Dynamics 365. Sign in to your Dataverse environment when prompted and select Allow.

2Pin the pane open

Select the pin icon in the Dynamics 365 pane to keep it open as you navigate between emails. Unpin to close it. Pinning is supported in both Outlook desktop and Outlook Web Access.

3View contact and account information

When you open an email, the Dynamics 365 pane shows information about the sender and recipients if they exist as contacts or leads in the CRM. Select a recipient to view their details, and expand to see their related account, opportunities, and other data.

4Track an email

To save a copy of the email in Dataverse, select Track in the Dynamics 365 pane. The email appears on the timeline of any linked records. The status changes to Track Successful when complete. Track Pending means the sync is in progress.

5Set Regarding to link to a record

Select Set Regarding in the pane to link the email to a specific CRM record — such as an opportunity, case, lead, account, or contact. This links the tracked email to that record's timeline. You can change or remove the regarding link at any time.

6Create a new CRM record

Use the navigation bar at the top of the pane and select the quick create (plus) icon to create a new record — such as a contact, lead, task, or phone call — without leaving Outlook. The new record is created in Dataverse immediately.

7Navigate using the site map

Select the site map (grid) icon in the navigation bar to access Home, Recent, Pinned, and Dashboards within the pane. Use Recent to find recently viewed CRM records. Use Pinned to bookmark records you check frequently.

Tracked emails appear on CRM timelines when server-side synchronisation is correctly configured and the mailbox has been approved and tested. If emails are not appearing, the admin should check the mailbox status for that user.
