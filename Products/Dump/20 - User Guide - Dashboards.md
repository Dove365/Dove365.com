# User Guide - Dashboards

## Purpose

Dashboards help users understand CRM activity and focus on work that needs attention.

## Dashboard Area

The app includes a Dashboard navigation group. Design context indicates:

- An HTML CRM data summary page.
- An activity dashboard for viewing activities across the CRM.

## Using Dashboards

1. Open Dashboard from the app navigation.
2. Review key cards, charts, or grids.
3. Open records that need follow-up.
4. Use views and filters where available.

## Known Note

The HTML Dashboard is a temporary place holder for an upcoming generative page. The
generative page may require further work or removal if deployment dependencies cause issues. If the page is unavailable, use the activity dashboard and table views until it is finalised.
