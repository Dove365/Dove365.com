# User Guide - Service Offerings

## Purpose

Service Offerings are the services your organisation sells or supports. They help categorise opportunities and cases consistently.

## Admin Maintenance

Admin users can create and maintain service offerings.

Recommended fields:

- Service Name
- Service Code
- Service Type
- Service Category
- Description
- Billing Type
- Currency
- Default Price
- Default Cost
- Duration Unit
- Default Duration

## Reporting Use

Consistent service offering use supports:

- Pipeline reporting by service.
- Case categorisation by service.
- Identification of frequently requested services.

## Scope Note

Default price, default cost, billing type, and duration fields are intended for opportunity-stage and service-categorisation use. They are not a full sales order, invoicing, or product catalogue system.
