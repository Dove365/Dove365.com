# User Guide - Cases

## Purpose

Cases track customer support requests, issues, service requests, complaints, billing queries, change requests, and general enquiries.

Known process flow: New -> Investigation -> Resolution -> Escalate, if needed -> Resolved.

Known statuses: New, Not Started, In Progress, Waiting on Customer, Waiting on Third Party, Delayed, Cancelled, Inactive.

## Create a Case

1. Open Service > Cases.
2. Select New.
3. Enter case name, account, contact, case type, priority, source, target resolution date, and details.
4. Save.
